## Headless CMS Api

### A New Way To Hook Into Group, Model or Entry Lifecycle

We removed existing `CmsContentModelHookPlugin` and `CmsContentEntryHookPlugin` support in favor of publish/subscribe pattern. Groups did not have lifecycle support, so we added those.

Available options for groups are:
 * onBeforeGroupCreate
 * onAfterGroupCreate
 * onBeforeGroupUpdate
 * onAfterGroupUpdate
 * onBeforeGroupDelete
 * onAfterGroupDelete

Available options for model are:
* onBeforeModelCreate
* onAfterModelCreate
* onBeforeModelUpdate
* onAfterModelUpdate
* onBeforeModelDelete
* onAfterModelDelete

And, in the end, for the entries:
 * onBeforeEntryCreate
 * onAfterEntryCreate
 * onBeforeEntryRevisionCreate
 * onAfterEntryRevisionCreate
 * onBeforeEntryUpdate
 * onAfterEntryUpdate
 * onBeforeEntryDelete
 * onAfterEntryDelete
 * onBeforeEntryRevisionDelete
 * onAfterEntryRevisionDelete
 * onBeforeEntryPublish
 * onAfterEntryPublish
 * onBeforeEntryUnpublish
 * onAfterEntryUnpublish
 * onBeforeEntryRequestChanges
 * onAfterEntryRequestChanges
 * onBeforeEntryRequestReview
 * onAfterEntryRequestReview


All the options are available on `context.cms` object. Here is an example on how to subscribe to afterEntryCreate:

```typescript
import { CmsContext } from "@webiny/api-headless-cms/types";
import { ContextPlugin } from "@webiny/handler/plugins/ContextPlugin";

const plugin = new ContextPlugin<CmsContext>(async context => {
    context.cms.onAfterEntryCreate.subscribe(async params => {
        const {
            /**
             * Input received from the user via GraphQL
             */
            input,
            /**
             * Entry created out of user input
             */
            entry,
            /**
             * The model this entry is going to belong to
             */
            model
        } = params;
        /**
         * Trigger your custom function that notifies some other system about new entry.
         */
        await notifyDifferentSystemAboutNewEntry({
            input,
            entry,
            mode
        });
    });
});
```
Of course, do not forget to register your plugin in the system.
